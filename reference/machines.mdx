## Overview

Bee Swarm Simulator features numerous machines and collectibles that provide resources, buffs, and items. The macro can automatically collect from these sources on configurable timers.

## Core Machines

<CardGroup cols={2}>
  <Card title="Blender" icon="blender">
    **Location:** Near hive  
    **Function:** Crafts smoothies and items  
    
    The blender combines ingredients to create powerful smoothies and items. The macro supports automated blending with configurable recipes.
    
    **Configuration:**
    - Up to 3 simultaneous blend recipes
    - Configurable amounts and repeat counts
    - Infinite repeat option available
    
    ```ini
    [Blender]
    BlenderCheck=1
    BlenderItem1=None
    BlenderItem2=None  
    BlenderItem3=None
    BlenderAmount1=0
    BlenderAmount2=0
    BlenderAmount3=0
    BlenderIndex1=1  # Repeat count (or "Infinite")
    BlenderIndex2=1
    BlenderIndex3=1
    ```
  </Card>

  <Card title="Wealth Clock" icon="clock">
    **Location:** Wealth Clock area  
    **Cooldown:** 1 hour  
    **Provides:** Honey bonus
    
    Collecting from the Wealth Clock provides a significant honey bonus.
  </Card>

  <Card title="Wind Shrine" icon="wind">
    **Location:** Sunflower/Dandelion area  
    **Function:** Donates items for buffs  
    
    Donate cloud vials or other items to receive powerful wind shrine buffs.
  </Card>

  <Card title="Ant Pass" icon="ticket">
    **Location:** Ant Gate  
    **Cooldown:** Variable  
    **Function:** Access to ant challenges  
    
    **Actions:**
    - **Pass** - Enter ant challenge
    - **Challenge** - Start ant challenge (uses tickets if enabled)
    
    ```ini
    [Collect]
    AntPassCheck=0
    AntPassAction=Pass  # or "Challenge"
    AntPassBuyCheck=0   # Use tickets
    ```
  </Card>

  <Card title="Robo Pass" icon="robot">
    **Location:** Robo area (beta)  
    **Cooldown:** Variable  
    **Function:** Access to robo challenges

```ini

NormalMemoryMatchCheck=0
MegaMemoryMatchCheck=0
ExtremeMemoryMatchCheck=0
NightMemoryMatchCheck=0
WinterMemoryMatchCheck=0
MemoryMatchInterruptCheck=0  # Allow gather interrupt
```

  </Card>

  <Card title="Sticker Stack" icon="sticker">
    **Location:** Sticker area  
    **Function:** Stack stickers for collection  
    
    **Mode Options:**
    - **Detect** - Auto-detect when stack is ready
    - **Timer** - Collect on fixed interval (900-86400 seconds)
    
    **Items:**
    - Tickets
    - Hive skins
    - Cub Buddy skins  
    - Vouchers
    
    ```ini
    [Stickers]
    StickerStackCheck=0
    StickerStackMode=0          # 0=Detect, 1=Timer
    StickerStackTimer=900       # Seconds (if timer mode)
    StickerStackItem=Tickets    # Target item
    StickerStackHive=0          # Collect hive skins
    StickerStackCub=0           # Collect cub skins
    StickerStackVoucher=0       # Collect vouchers
    ```
  </Card>

  <Card title="Sticker Printer" icon="printer">

    **Egg Types:**

## Collection Priority

The macro collects from machines in a specific order for optimal efficiency:

<Steps>
  <Step title="High Priority">
    - Wealth Clock (honey bonus)
    - Dispensers (free resources)
    - Blender (if recipes configured)
  </Step>
  
  <Step title="Medium Priority">
    - Memory Match games
    - Ant Pass / Robo Pass
    - Sticker machines
  </Step>
  
  <Step title="Low Priority / Event-Based">
    - Beesmas machines (seasonal)
    - Special events (Honey Storm)
    - Optional collectibles
  </Step>
</Steps>

## Automation Tips

<AccordionGroup>
  <Accordion title="Optimize Collection Timing">
    Enable only the machines you need to reduce unnecessary travel time. Disable seasonal machines when events are not active.
  </Accordion>

  <Accordion title="Blender Recipes">
    Configure up to 3 simultaneous blender recipes. Use the "Infinite" repeat option for continuous crafting of frequently-needed items.
  </Accordion>

  <Accordion title="Memory Match Strategy">
    Enable memory match gather interrupt only if you want to prioritize memory match over farming. This can reduce honey/pollen gains but provides valuable items.
  </Accordion>

  <Accordion title="Sticker Stack Detection">
    Use "Detect" mode for sticker stack to automatically collect when stacks are ready, rather than fixed timers which may miss or waste stacks.
  </Accordion>

  <Accordion title="Ant Pass Optimization">
    If running Bucko or Riley quests, enable Ant Pass with "Challenge" action and ticket usage for maximum quest completion efficiency.
  </Accordion>
</AccordionGroup>

## GUI Controls

The macro GUI provides controls for all collectible items:

- **Checkboxes** - Enable/disable each machine
- **Action selectors** - Choose specific actions (e.g., Ant Pass: Pass vs Challenge)
- **Item selectors** - Choose specific items (e.g., Sticker Stack: Tickets, Hive, Cub, Voucher)
- **Options buttons** - Access advanced settings (e.g., Memory Match options)
